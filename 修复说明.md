# MCP 服务器启动问题修复

## 问题描述

在 Cursor 中启动 MCP 服务器时出现以下错误：

```
RuntimeWarning: 'tree_sitter_analyzer_v2.mcp.server' found in sys.modules after import of package 'tree_sitter_analyzer_v2.mcp', but prior to execution of 'tree_sitter_analyzer_v2.mcp.server'; this may result in unpredictable behaviour

Server failed: unhandled errors in a TaskGroup (1 sub-exception)
```

## 问题原因

1. **模块导入警告**: 使用 `python -m tree_sitter_analyzer_v2.mcp.server` 时，Python 会先导入 `tree_sitter_analyzer_v2.mcp` 包，然后再执行 `server.py`，导致模块在 `sys.modules` 中的状态不一致。

2. **TaskGroup 错误**: MCP 服务器的初始化选项配置不正确，缺少必要的 `ServerCapabilities` 设置。

## 解决方案

### 修复 1: 添加 `__main__.py` 入口点

创建了 `tree_sitter_analyzer_v2/mcp/__main__.py` 文件：

```python
"""
MCP Server entry point for tree-sitter-analyzer v2.
"""

from tree_sitter_analyzer_v2.mcp.server import main_sync

if __name__ == "__main__":
    main_sync()
```

这样可以使用 `python -m tree_sitter_analyzer_v2.mcp` 来启动服务器，避免模块导入警告。

### 修复 2: 修正 ServerCapabilities

在 `server.py` 中添加了正确的 `ServerCapabilities` 配置：

```python
from mcp.server.models import ServerCapabilities
from mcp.types import ToolsCapability

# 在 run() 方法中
capabilities = ServerCapabilities(
    tools=ToolsCapability(listChanged=True),
)

options = InitializationOptions(
    server_name="tree-sitter-analyzer-v2",
    server_version="2.0.0-alpha.1",
    capabilities=capabilities,
)
```

## 技术细节

### 为什么需要 `__main__.py`

当使用 `python -m package.module` 时：
- Python 会先导入整个包
- 然后查找并执行指定的模块
- 如果模块中有 `if __name__ == "__main__"` 代码，可能会导致模块状态不一致

使用 `python -m package` 时：
- Python 会查找包中的 `__main__.py` 文件
- 直接执行该文件
- 避免了模块导入顺序问题

### ServerCapabilities 的作用

`ServerCapabilities` 告诉 MCP 客户端（如 Cursor）服务器支持哪些功能：
- `tools`: 支持工具调用
- `listChanged`: 支持工具列表变更通知

这是 MCP 协议规范的要求，缺少会导致初始化失败。
