# 实际使用场景发现的关键问题

## 🔴 P0 - 致命问题

### 问题 1: V2 MCP 工具全部超时
**场景**: 尝试使用任何 V2 MCP 工具分析代码
**现象**: 所有工具调用都返回 "Request timed out"
**影响**: V2 完全无法使用
**根本原因**: 
1. MCP 服务器可能未正确启动
2. 工具执行时间过长
3. 超时配置过短
4. 可能存在死锁或无限循环

**优先级**: P0 - 必须立即修复
**影响范围**: 100% V2 功能不可用

### 问题 2: analyze_code_graph 工具性能问题
**场景**: 分析单个文件或目录
**现象**: 即使单文件也超时
**影响**: 核心功能不可用
**根本原因**:
1. Tree-sitter 解析可能阻塞
2. 图构建算法效率低
3. 没有进度反馈
4. 缺少超时控制

**优先级**: P0 - 核心功能
**影响范围**: 代码地图、调用链分析

### 问题 3: find_files 工具超时
**场景**: 查找项目中的文件
**现象**: 简单的文件查找也超时
**影响**: 基础功能不可用
**根本原因**:
1. fd 工具调用可能阻塞
2. 大量文件时没有分页
3. 输出处理效率低

**优先级**: P0 - 基础功能
**影响范围**: 所有需要文件列表的操作

- 超时设置过短（默认 30s？）
- 需要检查 mcp.json 配置

- Tree-sitter 解析阻塞
- 文件 I/O 效率低

- Windows 路径处理
- 文件权限问题
- 进程通信问题

**验证方法**:
1. 直接运行 Python 代码（绕过 MCP）
2. 添加详细日志
3. 使用 profiler 分析
4. 测试小文件
