## 2. 基本的な使い方

### 2.1 単一ファイルのクエリ解析（検証済み）

**重要:** CLIはクエリまたはモード指定が必須です。

```bash
# Pythonファイルの関数を抽出
uv run tree-sitter-analyzer examples/sample.py --query-key function

# Javaファイルのクラスを抽出
uv run tree-sitter-analyzer examples/Sample.java --query-key class

# メソッドを抽出
uv run tree-sitter-analyzer examples/Sample.java --query-key method
```

**出力例（JSON形式）:**
```json
[
  {
    "capture_name": "function",
    "node_type": "function_definition",
    "start_line": 21,
    "end_line": 24,
    "content": "def __post_init__(self):\n    ..."
  }
]
```

---

### 2.2 高度な解析モード（検証済み）

```bash
# 詳細な要素情報と統計
uv run tree-sitter-analyzer examples/sample.py --advanced

# サマリー（クラスとメソッドの概要）
uv run tree-sitter-analyzer examples/sample.py --summary

# 構造解析
uv run tree-sitter-analyzer examples/sample.py --structure
```

**advanced出力例:**
```json
{
  "file_path": "examples/sample.py",
  "language": "python",
  "line_count": 256,
  "element_count": 27,
  "node_count": 1821,
  "elements": [...]
}
```

---

### 2.3 テーブル形式での表示（検証済み）

**重要:** `--table`と`--query-key`は**併用できません**。

```bash
# テーブル形式（ファイル全体の要素を表示）
uv run tree-sitter-analyzer examples/sample.py --table compact

# フル情報のテーブル
uv run tree-sitter-analyzer examples/sample.py --table full

# CSV形式（スプレッドシート用）
uv run tree-sitter-analyzer examples/Sample.java --table csv

# JSON形式のテーブル
uv run tree-sitter-analyzer examples/sample.py --table json
```

**注意事項:**
- `--table`は単独で使用（クエリキー不要）
- ファイル全体の構造をテーブル形式で表示
- `--query-key`との併用は不可（エラーになります）

---

### 2.4 Pythonスクリプトから使用

```python
from tree_sitter_analyzer import analyze_file

# ファイルを解析
result = analyze_file('src/main.py')

# メトリクス取得
print(f"総行数: {result['line_count']}")
print(f"要素数: {len(result['elements'])}")

# 詳細情報
for element in result['elements']:
    print(f"{element['element_type']}: {element['name']}")
```

**設定ファイル編集:** `~/.config/claude/claude_desktop_config.json` (Linux/Mac) または `%APPDATA%\Claude\claude_desktop_config.json` (Windows)

```json
    }
  }
```

**注:** `tree-sitter-analyzer-mcp`は`pyproject.toml`で定義されたMCPサーバーのエントリーポイントです。

```
        - コード行数: 180行

```

## 6. トラブルシューティング

### 6.1 よくある問題と解決法

#### 問題1: "Please specify a query or --advanced option"

**原因:** クエリやモード指定なしでファイルを指定

**解決策:**
```bash
# NG: uv run tree-sitter-analyzer file.py
# OK: 以下のいずれか
uv run tree-sitter-analyzer file.py --query-key function
uv run tree-sitter-analyzer file.py --advanced
uv run tree-sitter-analyzer file.py --summary
uv run tree-sitter-analyzer file.py --table compact
```

---

#### 問題2: "--table and --query-key cannot be used together"

**原因:** `--table`と`--query-key`を同時に指定

**解決策:**
```bash
# NG: uv run tree-sitter-analyzer file.py --query-key function --table compact
# OK: どちらか一方のみ
uv run tree-sitter-analyzer file.py --table compact
uv run tree-sitter-analyzer file.py --query-key function

# フィルタ使用（--query-keyと併用可能）
uv run tree-sitter-analyzer file.java --query-key method --filter "name=main"
```

---

#### 問題3: 言語が検出されない

```bash
# 言語を明示的に指定
uv run tree-sitter-analyzer file.txt --language python --query-key function
```

---

#### 問題4: プロジェクトルート外のファイルエラー

**原因:** セキュリティ制限

**解決策:**
```bash
# プロジェクトルートを指定
uv run tree-sitter-analyzer file.py --project-root /path/to/project --query-key function
```

---

#### 問題5: クエリ結果が"No results found"

**原因:** ファイルに該当する構造が存在しない

**例:**
```bash
# Sample.javaにはSpringアノテーションが存在しない
uv run tree-sitter-analyzer examples/Sample.java --query-key spring_controller
# INFO: No results found matching the query.
```

**対処法:** 
- 該当する構造を持つファイルを指定
- `--list-queries`で利用可能なクエリを確認
- `--advanced`でファイル全体を解析して構造を確認
