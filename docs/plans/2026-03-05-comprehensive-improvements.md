## 🎯 代码质量改进

### 1. 拆分 God类

**问题**: sql_plugin.py (2462行) 和 markdown_plugin.py (1973行) 过于庞大

**解决方案**:
```
sql_plugin.py/
├── __init__.py
├── base.py          # 基础类
├── extractor.py     # 提取逻辑
├── formatter.py     # 格式化逻辑
├── queries.py       # Tree-sitter 查询
└── utils.py         # 工具函数
```

### 2. 消除重复代码

**问题**: 多个语言插件有相似的提取模式

**解决方案**:
- 创建 `BaseLanguagePlugin` 抽象类
- 提取公共方法到 `LanguagePluginMixin`
- 使用模板方法模式

### 3. 类型注解完善

**问题**: 部分函数缺少返回类型

**解决方案**:
- 添加完整的类型注解
- 使用 `TypeVar` 和 `Generic`
- 启用更严格的 mypy 配置
