# 动态命令接口 — 执行流程

本文档描述与「动态命令接口系统」规则一致的执行循环，供在对话中按步骤执行。

---

## 单轮循环

每一轮按顺序做三件事：

1. **执行当前任务**  
   完成你在对话里要求的操作（编码、测试、修改配置等）。

2. **进入输入阶段**  
   在聊天附带的终端中执行规则所规定的命令，使终端进入“等待输入”状态。  
   用户可输入多行内容，以**空行**结束；输入内容将作为下一轮的指令或数据。

3. **根据输入决定下一步**  
   - 若用户输入为 **`stop`**（不区分大小写）：结束流程，不再继续循环。  
   - 否则：把本次输入当作新的指令或上下文，从步骤 1 开始下一轮（执行新任务 → 再次进入输入阶段 → 再根据输入继续）。

---

## 何时启动输入阶段

- 在「执行当前任务」之后、需要用户给出下一步指示时，启动输入阶段。
- 若用户已明确说“就做到这里”“不用再问”等，可不再启动输入阶段，直接结束。

---

## 流程图（简要）

```
[ 执行你指定的操作 ]
        ↓
[ 在终端运行规则规定的输入采集命令 ]
        ↓
[ 用户输入多行，空行结束 ]
        ↓
  输入 == "stop" ?  ——是→ [ 结束流程 ]
        ↓ 否
[ 用输入作为新指令，回到「执行操作」]
```

---

## 注意

- 输入阶段使用的脚本由规则定义，存在时不可修改，亦不在对话中讨论其实现。
- 确保在**聊天内的终端**中运行输入命令，以便在同一会话中捕获用户输入并继续后续操作。
